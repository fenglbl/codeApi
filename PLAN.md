# CodeAip PLAN

## 1. 项目目标

CodeAip 是一个本地 OpenAI 兼容 API 网关，目标是把多个第三方 OpenAI 格式上游统一收口到本地地址，并提供一个后台管理系统做配置、路由和密钥管理。

### 一期核心目标

1. 提供本地统一 API 入口：
   - `GET /v1/models`
   - `POST /v1/chat/completions`
   - `POST /v1/embeddings`
2. 提供后台管理页面：
   - 登录认证
   - 上游管理
   - 本地 API Key 管理
3. 支持多个 OpenAI 兼容上游：
   - 配置 base URL
   - 配置 API Key
   - 配置模型列表
   - 启用 / 禁用
4. 支持多个本地 API Key：
   - 每个本地 Key 可绑定多个上游
   - 每个本地 Key 可设置默认上游
   - 支持本地模型名 -> 上游模型名映射
5. 安全要求：
   - 上游 API Key 加密存储
   - 本地 API Key 支持校验
   - 管理后台需要登录/JWT 保护

## 3. 当前技术方案

## 3.1 技术栈

- 前端：Vue 3 + Vite + Pinia + Axios + Element Plus
- 后端：Node.js + Express
- 数据库：MongoDB

## 3.2 项目结构

- 前端：`/www/wwwroot/nodejs/codeAip/admin`
- 后端：`/www/wwwroot/nodejs/codeAip/api`
- 需求草稿：`/www/wwwroot/nodejs/codeAip/new.md`
- 本计划：`/www/wwwroot/nodejs/codeAip/PLAN.md`

## 3.3 路由设计

### 对外接口
- `/v1/models`
- `/v1/chat/completions`
- `/v1/embeddings`

### 后台页面
- `/admin`

### 后台接口
- `/api/admin/auth/*`
- `/api/admin/upstreams/*`
- `/api/admin/local-keys/*`

### 健康检查
- `/health`

## 3.4 安全设计

### 上游 API Key
- 明文不直接存储
- 使用服务端加密后入库
- 加密主密钥放 `.env`

### 本地 API Key
- 当前方案已支持：
  - `keyHash`：用于校验请求
  - `rawKeyEncrypted/rawKeyIv`：用于后台管理员查看/复制明文
- 页面默认脱敏显示
- 仅管理员登录后按需查看明文

### 管理后台
- 用户名密码登录
- JWT 鉴权

## 6. 当前主要风险

### 风险 1：服务常驻问题
当前开发过程中，临时拉起的 node 进程容易退出或端口冲突，需要一个稳定的正式启动方案。

### 风险 2：权限混乱
`/www/wwwroot/nodejs/codeAip` 曾出现不同属主导致的 EACCES 问题，后续必须统一运行用户和构建用户。

### 风险 3：真实上游联调尚未闭环
虽然后台和代理代码已基本搭起，但还需要拿真实上游做完整通路验证。

### 风险 4：本地 Key 明文可查看的安全权衡
为了满足后台“显示/复制”需求，本地 Key 明文采取加密存储而不是完全不可逆，这需要依赖后台鉴权和服务端密钥管理。
